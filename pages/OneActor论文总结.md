# OneActor: Consistent Subject Generation via Cluster-Conditioned Guidance

> NeurIPS 2024 | 西交大 + 中电信AI Lab | [arXiv](https://arxiv.org/abs/2404.10267) | [GitHub](https://github.com/JoHnneyWang/OneActor)

---

## 1. 问题定义

文本生成图像扩散模型如 SDXL 可以生成高质量图像，但其**随机采样**导致同一提示词生成的多张图像中角色身份不一致。现有方法要么依赖外部参考图像（Personalization），要么需要昂贵的全模型调优（TheChosenOne），要么推理时间翻倍（ConsiStory）。

OneActor 的目标：**仅靠文本提示词驱动**，经过一次轻量调优后，能为同一主体生成在任意新提示词下保持身份一致的多张图像。

---

## 2. 核心思想

### 2.1 聚类形式化

在扩散模型的潜在空间中，样本天然形成层级聚类结构：

```
"hobbit" 基础簇（同一类别的所有图像）
  ├── 子簇 A（高瘦尖耳）
  ├── 子簇 B（矮胖圆耳）
  ├── 子簇 C（某特定长相）
  └── 子簇 D（另一种特定长相）
```

普通扩散模型生成的多张图像会随机散落在不同子簇 → **身份不一致**。OneActor 的目标是引导去噪轨迹始终落入**同一个目标子簇**。

### 2.2 聚类引导评分函数

从概率角度出发，期望分布为：

```
p(x) · p(S^tar | x) / Π p(S^aux_i | x)
```

取负对数梯度，借助重参数化技巧，得到核心公式 (Eq.6)：

```
ε_final = ε_θ(z_t, t)                          ← 无条件扩散先验
        + η₁·[ε_θ(z_t,t,S^tar) - ε_θ(z_t,t)]     ← 目标吸引分数
        - η₂·Σ[ε_θ(z_t,t,S^aux_i) - ε_θ(z_t,t)]  ← 辅助排斥分数
```

两个力方向相反：η₁ 控制"锁定目标身份"的强度，η₂ 控制"排斥非目标身份"的强度。

## 3. 方法架构

### 3.1 整体流程

```
┌──────────────────────────────────────────────────────┐
│  ① 数据生成 (generate_data.py, ~30s)                 │
│     p^tar → 普通SDXL生成N张图 → 用户选1张作目标      │
│           → 其余N-1张作辅助集                        │
│           → 顺带存下U-Net编码器中间特征h              │
├──────────────────────────────────────────────────────┤
│  ② 投影器调优 (tune.py, 3-6min)                     │
│     仅训练ϕ (ResNet + Linear + AdaIN)                │
│     U-Net/VAE/文本编码器全部冻结                      │
├──────────────────────────────────────────────────────┤
│  ③ 推理 (inference.py)                              │
│     ϕ输出c_Δ → 偏移基准词嵌入 → U-Net去噪轨迹        │
│     被拐向目标子簇 → 生成身份一致的角色              │
└──────────────────────────────────────────────────────┘
```

### 3.2 聚类条件模型

**支持网络 ϕ** 是唯一可训练组件：

```
ϕ(h, c) → c_Δ    （一个语义方向向量）

其中 h = E_u(z₁, c)  ← U-Net编码器中间特征（生成数据时顺带存储）
      c = E_t(p)      ← 文本编码器输出
```

| 设计选择 | 动机 | 收益 |
|---------|------|------|
| 用 z₁ 近似替代 z₀ | 避免额外的 U-Net 前向传播 | **节省 30% 计算量** |
| 复用 U-Net 编码器 E_u 作特征提取 | 无需训练特征提取器 | 零额外参数 |
| 投影器只含 ResNet+Linear+AdaIN | 容量限制即正则化 | 防过拟合 |

### 3.3 词级偏移

```
提示词嵌入序列: c = [c_a, c_hobbit, c_with, c_robes]
                                         ↓ 仅修改基准词嵌入
                        c_hobbit' = c_hobbit + c_Δ
                                         ↓
新序列:              c' = [c_a, c_hobbit', c_with, c_robes]
```

交叉注意力机制天然将 `c_hobbit` 与角色空间区域绑定，因此修改作用自动限定在角色区域，无需显式分割。

### 3.4 辅助样本泛化调优

每步训练的 batch：

```
B = {x^tar, h^tar} ∪ {x^aux_i, h^aux_i}_{i=1}^K
     1个目标样本          K个辅助样本
```

三种损失函数：

| 损失 | 公式 | 作用 |
|------|------|------|
| L_tar | ‖ε − ε_{θ,ϕ}(z_t^tar, t, c, ϕ(h^tar, c))‖² | 学会重建目标身份 |
| L_aux | ‖ε − ε_{θ,ϕ}(z_t^aux, t, c, ϕ(h^aux, c))‖² | 提供"负样本"，防止过拟合 |
| L_aver | ‖ε − ε_{θ,ϕ}(z_t^aver, t, c, c_Δ^aver)‖² | 学习平均辅助条件，推理时用作排斥锚点 |

平均辅助条件：`c_Δ^aver = 1/K · Σ ϕ(h_i^aux, c)`

总损失：`L = L_tar + λ₁·L_aux + λ₂·L_aver`

三种反向传播策略完全一致（标准扩散 MSE），对比效果通过数据组合而非损失函数设计实现。

### 3.5 通用性来源

| 层面 | 做法 | 效果 |
|------|------|------|
| 架构冻结 | U-Net/VAE/文本编码器全部冻住 | 保留全部扩散先验与多样性 |
| 模板化训练 | 随机模板填充基准词 ("a portrait of a hobbit", "a hobbit in garden", ...) | ϕ 学会忽略文字变化，只提取身份 |
| 辅助样本 | 同一提示词生成的不同 hobbit 作反例 | ϕ 学会区分"同类"与"同身份" |
| 投影器容量 | 极小参数量 (ResNet+Linear+AdaIN) | 容量限制本身是正则化 |

### 3.6 推理策略

| 策略 | 做法 | 效果 |
|------|------|------|
| 语义插值 | c_Δ = v·c_Δ^target, v∈[0,1] | v=1 完全一致，v<1 增加多样性 |
| 簇引导尺度 | 调节 η₁ 和 η₂ | η₁ 锁定身份，η₂ 排斥非目标 |
| 部分步引导 | 只在部分去噪步施加引导 | 解耦一致性与多样性 |
| 零开销模式 | 设 η₂=0 | 不额外增加推理时间 |
| 平均条件近似 | 用 c_Δ^aver 替代逐个计算 N-1 个辅助条件 | 推理从 N 次 U-Net 前向降为 2 次 |

语义插值性质的**理论贡献**：首次证明扩散模型的语义空间与潜在空间具有相同的插值/外推属性，为精细生成控制提供了新工具。

### 3.7 多主体生成

| | 变体一（联合分布） | 变体二（独立训练） |
|---|---|---|
| 投影器数量 | 1 个 | 每主体 1 个 |
| 训练方式 | 多主体同框图一次训 | 各主体单独训 |
| 增量添加主体 | 不可 | 可以 |
| 空间约束 | 靠交叉注意力自动 | 注意力图 + 空间掩码 |

## 4. 实验评估

### 4.1 定量指标

| | TheChosenOne 设置 | ConsiStory 设置 |
|---|---|---|
| 身份一致性 | **CLIP-I**（生成图与目标图的 CLIP 图像嵌入余弦相似度） | **DreamSim**（专注前景主体，对人眼感知更贴合） |
| 提示符合度 | **CLIP-T**（提示词文本嵌入与生成图图像嵌入相似度） | **CLIP-Score**（无参考评估） |

测试数据由 ChatGPT 自动生成，避免人工 cherry-picking。

### 4.2 对比方法

| 类别 | 方法 | 弱点 |
|------|------|------|
| 调优式个性化 | Textual Inversion | 缺乏一致性 |
| 调优式个性化 | DreamBooth | 提示符合度低 |
| 编码式个性化 | IP-Adapter | 偶尔不稳定 |
| 编码式个性化 | BLIP-Diffusion | 图像质量差 |
| 编码式个性化 | ELITE | 多样性有限 |
| 一致性生成 | TheChosenOne | 全模型调优代价大 |
| 一致性生成 | ConsiStory | 多主体布局单一 |

### 4.3 核心结果

- **Pareto 前沿**：在身份一致性 vs 提示符合度的散点图中，OneActor 在两套指标设置下均形成新的 Pareto 前沿，即没有其他方法在两个维度同时超越它
- **消融实验**：
  - 去掉 L_aver → 一致性下降
  - 去掉 L_aux + L_aver → 严重过拟合
- **效率**：调优 3-6 分钟（4× faster than DreamBooth），推理可选零开销
- **t-SNE 验证**：加 cluster guidance 后，生成样本的潜在码明显聚集在目标样本周围
- **应用**：故事书创作、风格迁移、ControlNet 位姿控制

---

## 5. 关键贡献总结

1. **首次将一致性主体生成形式化为聚类引导问题**，推导出"目标吸引 + 辅助排斥"的评分函数
2. **仅调语义空间（ϕ）**，不碰扩散主干，4× 更快调优，可选零推理开销
3. **辅助样本泛化调优**解决 one-shot 过拟合，用数据组合替代专门对比损失函数
4. **首次证明语义空间的插值性质**，为精确生成控制提供新工具
5. **支持多主体生成和增量添加**，兼容 ControlNet 等工作流

---

## 6. 局限性

- 每次换角色需重新调优 3-6 分钟（虽然已有预训练通用 ϕ 的理论可能，但未开源实现）
- 对极端夸张的虚构角色或复杂多对象交互场景可能失效
- 依赖用户手动指定基准词，不能自动检测主语
- 需要 A100 GPU，消费级显卡可能更慢

---

## 7. 可迁移研究方向

1. **3D 一致性生成**：将 cluster-conditioned 范式扩展到 NeRF / 3D Gaussian Splatting
2. **视频一致性**：时序聚类引导替代帧间特征匹配
3. **多模态条件**：在语义空间融合文本 + 图像 + 草图条件
4. **预训练通用 ϕ**：大规模数据预训练后 zero-shot 推理，消除逐角色调优
5. **自动基准词检测**：用 NLP 或注意力分析自动定位主语词
